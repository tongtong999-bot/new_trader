# 交互式输入问题清单（云端部署需修复）

## 发现的交互点

### 1. `live_trading_v52.py` - `main()` 函数

**位置**: 第897行、第911行

**交互内容**:
- 选择运行模式（1-4）：
  - 1. 使用当前配置启动（OKX模拟交易）
  - 2. 切换到本地模拟盘（不下单）
  - 3. 修改交易对
  - 4. 退出
- 输入交易对（如果选择3）

**影响**: 在 Docker/云端部署时会阻塞，等待用户输入

### 2. `multi_symbol_trading.py` - `main()` 函数

**位置**: 第202行、第211行、第239行

**交互内容**:
- 选择币种列表（1-3）：
  - 1. 使用默认币种列表启动
  - 2. 自定义币种列表
  - 3. 退出
- 循环输入币种（如果选择2）
- 确认启动（y/n）

**影响**: 在 Docker/云端部署时会阻塞，等待用户输入

## 解决方案

需要添加：
1. 命令行参数支持（`argparse`）
2. 环境变量支持（用于 Docker Compose）
3. 非交互模式（默认使用配置，跳过所有 `input()`）

## 修复优先级

- **高**: `live_trading_v52.py`（单币种，更常用）
- **高**: `multi_symbol_trading.py`（多币种，也需要修复）
